================================================================================
  SpatioThematicVLM Agent 评估体系说明
  —— 无 Ground Truth 的自动化验证与敏感性分析框架
================================================================================


一、设计理念
──────────────────────────────────────────────────────────────────

本评估体系针对"基于视觉语言模型的多智能体专题地图分析系统"设计，核心约束：
  - 无法获取标注好的 Ground Truth 数据
  - 无法进行专家打分

因此，全部指标均基于系统输出的结构特性、跨运行一致性、流水线内部自洽性
进行衡量，不依赖任何外部参考标准。

框架围绕 5 个研究问题（RQ）展开，共设计 16 个具体指标。


二、五个研究问题与指标体系
──────────────────────────────────────────────────────────────────

┌─────────────────────────────────────────────────────────────┐
│  RQ1: 结构化输出有效性（Schema Compliance）                  │
│  核心问题：VLM 能否稳定地按要求输出合法的结构化 JSON？         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  指标 1: JSON Parseable Rate（JSON 可解析率）                │
│    - 计算方式：VLM 原始文本能否被 json.loads 成功解析         │
│    - 值域：0% ~ 100%                                       │
│    - 意义：衡量模型对"只输出 JSON"这一指令的遵循程度          │
│                                                             │
│  指标 2: Pydantic Valid Rate（Pydantic 校验通过率）          │
│    - 计算方式：解析后的 JSON 能否通过对应 Pydantic 模型校验   │
│    - 值域：0% ~ 100%                                       │
│    - 意义：字段名、数据类型、必填项是否全部正确               │
│                                                             │
│  指标 3: Fallback Trigger Rate（兜底触发率）                 │
│    - 计算方式：输出中是否包含"解析失败"等预设失败标记         │
│    - 值域：0% ~ 100%                                       │
│    - 意义：越低越好，反映实际可用输出的比例                   │
│    - 检测关键词："解析失败"、"模型输出不符合Schema"、          │
│                  "analysis failed"                          │
│                                                             │
│  指标 4: Field Completeness（字段完整度）                    │
│    - 计算方式：已有效填充的字段数 / Pydantic Schema 总字段数  │
│    - 值域：0 ~ 1                                           │
│    - "有效填充"定义：值不为 None、空字符串、空列表、空字典     │
│    - 分别对 Visual Agent（7 个字段）和                       │
│      Semantic Agent（5 个字段）计算                          │
│                                                             │
│  附加指标: Output Richness（输出丰富度）                     │
│    - 各字段的文本长度、列表元素个数                           │
│    - 用于辅助判断输出是模板化填充还是真正有信息量的分析        │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  RQ2: 跨运行稳定性（Cross-run Consistency）                  │
│  核心问题：同一张图多次分析，核心结论是否一致？               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  指标 5: Jaccard Similarity（Jaccard 文本相似度）            │
│    - 计算方式：两次运行中同一文本字段的词集交并比             │
│    - 分词策略：中文按字切分，英文按词切分                     │
│    - 值域：0 ~ 1（1 = 完全一致）                            │
│    - 应用字段：theme、map_title、overall_summary、           │
│                main_theme 等                                │
│                                                             │
│  指标 6: Unique Values Count（唯一值计数）                   │
│    - 计算方式：同一字段在 N 次运行中产生了多少不同的值        │
│    - 值域：1 ~ N                                            │
│    - 越接近 1 表示该字段输出越稳定                           │
│                                                             │
│  指标 7: Output Richness Std（输出丰富度标准差）             │
│    - 计算方式：各次运行中总文本长度的标准差                   │
│    - 用于评估输出量是否存在大幅波动                          │
│                                                             │
│  对于列表类字段（如 legend、key_insights）：                  │
│    - 额外统计元素数量的 mean / std / range                   │
│                                                             │
│  对于数值类字段：                                            │
│    - 额外计算变异系数 CV = std / |mean|                     │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  RQ3: 生成参数敏感性（Parameter Sensitivity）                │
│  核心问题：模型生成参数和 Prompt 策略如何影响输出质量？       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  A. 三个生成参数的扫描：                                     │
│                                                             │
│    参数 1: temperature（采样温度）                           │
│      - 扫描值（standard 模式）：0.1, 0.3, 0.6, 0.8, 1.0    │
│      - 预期趋势：低温度 → 高合规率但内容模板化               │
│                   高温度 → 内容丰富但 JSON 格式可能出错       │
│                                                             │
│    参数 2: top_p（核采样概率）                               │
│      - 扫描值：0.5, 0.7, 0.85, 0.95                        │
│      - 预期趋势：与 temperature 类似的权衡                   │
│                                                             │
│    参数 3: max_new_tokens（最大生成长度）                    │
│      - 扫描值：128, 256, 512, 768, 1024                     │
│      - 预期趋势：过短 → JSON 被截断导致解析失败              │
│                   过长 → 可能引入冗余内容但合规率不受影响      │
│                                                             │
│    每个参数值的观测指标（3 个）：                             │
│      ① Schema 合规率                                        │
│      ② 总 Pipeline 延迟（秒）                               │
│      ③ 输出丰富度（总文本长度）                             │
│                                                             │
│  B. Prompt 变体敏感性：                                     │
│                                                             │
│    变体 1: baseline — 当前默认 prompt（含 CRITICAL RULES）   │
│    变体 2: minimal  — 仅保留 JSON Schema，去掉所有规则说明   │
│    变体 3: few_shot — 在 prompt 中加入一个 JSON 输出示例     │
│    变体 4: chinese  — 将整个系统 prompt 翻译为中文           │
│                                                             │
│    每个变体的观测指标（4 个）：                               │
│      ① JSON 可解析率                                        │
│      ② Pydantic 通过率                                      │
│      ③ 输出长度                                             │
│      ④ 推理延迟                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  RQ4: 输入鲁棒性（Input Robustness）                        │
│  核心问题：图片质量变化对系统分析结果影响多大？               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  10 种图片扰动方式：                                        │
│                                                             │
│   类别         扰动名称           参数设定                   │
│   ─────────   ──────────────     ────────────────           │
│   基准         original           原图不变                   │
│   分辨率       resize_50          缩小至 50%                 │
│   分辨率       resize_200         放大至 200%                │
│   压缩         jpeg_q20           JPEG quality=20（重度）    │
│   压缩         jpeg_q50           JPEG quality=50（中度）    │
│   模糊         blur               高斯模糊 radius=3         │
│   噪声         noise              高斯噪声 σ=15             │
│   光照         brightness_low     亮度系数 0.5（偏暗）       │
│   光照         brightness_high    亮度系数 1.5（偏亮）       │
│   信息丢失     crop_center_80     中心裁切保留 80%           │
│                                                             │
│  每种扰动下的观测指标（2 个）：                              │
│    ① Schema 合规率                                          │
│    ② Pipeline Coherence（流水线一致性综合分）                │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  RQ5: 多智能体流水线一致性（Inter-Agent Coherence）          │
│  核心问题：流水线各层输出之间是否内在自洽？                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  子指标 8:  Visual↔Semantic Theme Overlap                   │
│    - Visual Agent 的 theme 与 Semantic Agent 的 main_theme  │
│      之间的 Jaccard 相似度                                   │
│    - 权重：0.25                                             │
│                                                             │
│  子指标 9:  Visual Summary → Report Reference               │
│    - overall_summary 文本与最终 report 的 ROUGE-L F1        │
│    - 权重：0.15                                             │
│                                                             │
│  子指标 10: Key Insights → Report Reference                 │
│    - 每条 key_insight 与 report 的 ROUGE-L F1 取均值        │
│    - 权重：0.20                                             │
│                                                             │
│  子指标 11: Anomalies → Report Reference                    │
│    - 每条 anomaly.description 与 report 的 ROUGE-L F1      │
│      取均值；无异常时默认 1.0                                │
│    - 权重：0.15                                             │
│                                                             │
│  子指标 12: Map Title → Report Reference                    │
│    - 若 map_title 精确出现在 report 中 → 1.0               │
│    - 否则回退到 Jaccard 相似度                              │
│    - 权重：0.25                                             │
│                                                             │
│  综合分: Overall Coherence                                  │
│    = Σ(子指标 × 权重)                                       │
│    = 0.25×子指标8 + 0.15×子指标9 + 0.20×子指标10            │
│      + 0.15×子指标11 + 0.25×子指标12                        │
│    - 值域：0 ~ 1                                           │
│                                                             │
│  ROUGE-L F1 计算方法：                                      │
│    - 基于最长公共子序列（LCS）                               │
│    - Precision = LCS / len(text1)                           │
│    - Recall    = LCS / len(text2)                           │
│    - F1 = 2 × P × R / (P + R)                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  辅助维度: 性能效率（Latency）                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  按节点统计推理延迟：                                        │
│    - Visual Agent / Semantic Agent / Anomaly Agent /         │
│      Report Agent 各自的 mean、std、min、max、median         │
│    - 总 Pipeline 耗时                                       │
│                                                             │
│  用于分析：                                                  │
│    - 各节点的计算瓶颈                                        │
│    - 参数变化对推理速度的影响                                │
└─────────────────────────────────────────────────────────────┘


三、阈值选取说明
──────────────────────────────────────────────────────────────────

本框架有意不设置"通过/不通过"的硬阈值。原因：
  1. 缺乏 Ground Truth，无法定义绝对的"正确"
  2. 课程论文更关注趋势分析而非二元判定
  3. 连续分值提供的信息量大于离散等级

框架中存在的隐性阈值及其依据：

  ┌────────────────────────┬────────────┬──────────────────────┐
  │ 位置                    │ 阈值        │ 依据                  │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ Fallback 检测关键词     │ 3 个字符串   │ 与 nodes.py 中        │
  │                        │            │ _extract_and_validate │
  │                        │            │ _json 的 fallback     │
  │                        │            │ 输出一致              │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ Report 最小长度         │ 100 字符    │ nodes.py 中           │
  │ (report_agent_node)    │            │ report_agent_node     │
  │                        │            │ 原有设定              │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ 高斯噪声 σ             │ 15         │ CV 领域中等噪声       │
  │                        │            │ 标准设定              │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ 高斯模糊 radius        │ 3          │ 中等模糊，参考        │
  │                        │            │ ImageNet-C 基准       │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ JPEG 压缩质量           │ 20 / 50    │ 20 = 重度（可见伪影） │
  │                        │            │ 50 = 中度（轻微损失） │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ 亮度增强系数            │ 0.5 / 1.5  │ ±50% 亮度变化，      │
  │                        │            │ 模拟欠/过曝光         │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ 中心裁切比例            │ 80%        │ 各边裁 10%，模拟      │
  │                        │            │ 边缘图例信息丢失      │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ Coherence 权重分配      │ 见 RQ5     │ Title 和 Theme 为     │
  │                        │            │ 最核心分析结论，      │
  │                        │            │ 赋予最高权重 0.25     │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ temperature 扫描范围    │ 0.1 ~ 1.0  │ 覆盖从确定性输出到    │
  │                        │            │ 高随机性的完整区间    │
  ├────────────────────────┼────────────┼──────────────────────┤
  │ max_new_tokens 范围     │ 128 ~ 1024 │ 128 = 最短可能完整    │
  │                        │            │ JSON；1024 = 充裕上限 │
  └────────────────────────┴────────────┴──────────────────────┘


四、实验规模汇总（standard 模式）
──────────────────────────────────────────────────────────────────

  实验                    VLM 调用次数       预计耗时
  ───────────────────    ──────────────    ──────────
  一致性测试 (5 runs)     20 次             ~15 分钟
  Temperature 扫描        40 次             ~30 分钟
  top_p 扫描              32 次             ~25 分钟
  max_tokens 扫描         40 次             ~30 分钟
  图片扰动 (10 种)        40 次             ~30 分钟
  Prompt 变体 (4 种)       8 次             ~8 分钟
  ──────────────────────────────────────────────────
  合计                   ~180 次            ~2.5 小时
